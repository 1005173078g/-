# 自动化运行说明

## GitHub Pages

仓库使用 `main` 分支作为静态页面来源。每日页面保存在 `daily/YYYY-MM-DD.html`，公开链接格式为：

`https://1005173078g.github.io/-/daily/YYYY-MM-DD.html`

如果链接无法访问，请在 GitHub 仓库 Settings -> Pages 中开启 Pages，并选择 `main` 分支。

## 企业微信 Webhook

Webhook 不写入仓库。运行前在 PowerShell 设置：

```powershell
$env:WECOM_WEBHOOK_URL="你的企业微信群机器人 Webhook"
```

## 手动运行

准备 `data/YYYY-MM-DD.json` 后执行：

```powershell
& "G:\gj\python\python.exe" -m src.news_lianbo_daily.cli --date 2026-07-09 --send-wecom
```

也可以使用项目脚本：

```powershell
.\scripts\run_daily.ps1 -Date 2026-07-09 -SendWecom
```

## GitHub Actions 每日自动化

自动化文件：`.github/workflows/news-lianbo-daily.yml`

每天北京时间 21:40 自动运行。对应 GitHub Actions 的 UTC 时间是每天 13:40：

```yaml
cron: "40 13 * * *"
```

也可以在 GitHub 仓库的 Actions 页面手动点击 `workflow_dispatch` 运行。

### 必须设置的 Secrets

在 GitHub 仓库 Settings -> Secrets and variables -> Actions -> Repository secrets 中添加：

- `OPENAI_API_KEY`：用于生成结构化日报。
- `WECOM_WEBHOOK_URL`：企业微信群机器人 Webhook。

可选变量：

- `OPENAI_MODEL`：默认使用 `gpt-5.4-mini`。

### 自动化流程

1. 计算北京时间日期 `YYYY-MM-DD`。
2. 通过 OpenAI Responses API 查找并只基于央视网/CCTV 官方《新闻联播》来源生成结构化 JSON。
3. 写入 `data/YYYY-MM-DD.json`。
4. 渲染 `daily/YYYY-MM-DD.html`。
5. 执行测试。
6. 提交并推送当天 JSON 和 HTML。
7. 推送成功后发送企业微信群消息。
8. 如果生成、测试或发布失败，发送中文失败说明。

自动化运行时遵守：

- 所有输出使用中文。
- 不把企业微信 Webhook 写入仓库。
- 不使用 `推荐`、`买入`、`确定性机会` 等投资推荐措辞。
- 页面底部保留“不构成投资建议”的免责声明。
